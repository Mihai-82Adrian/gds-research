GDS Simulation Report

Phase 1: Synthetic World Generation

  • Synthetic world schema loaded.
  • Generated 14 semantic particles in memory.

Phase 2: Graph Construction & Reasoning

  • Semantic graph built successfully with 14 nodes.

Step 1: Initial Reasoning Query

  • Query: Find the cheapest path from ‘king’ to ‘power’ with an empty Overlay.
  • Result: Path "king -> power" found.
  • Initial Cost: 1.5102
    • Initial Cost Breakdown: inv_m0=0.9102, vad=0.0000, rel=0.6000, overlay=0.0000

Step 2: Internal Coherence Scoring

  • Goal: Evaluate which path is semantically richer based on the sum of concept mass (m0).
  • Coherence score for king -> power: 2.8904
  • Coherence score for king -> crown -> power: 3.9890
  • Conclusion: The path through ‘crown’ is considered semantically richer.

Step 3: Autonomous Learning Event

  • Action: Reinforce the richer path and penalize the alternative.
  • Learning Parameters: eta_pos (reinforcement) = -0.5, eta_neg (penalty) = 0.01, post_diffusion = disabled.
  • Outcome: 3 edges were updated in the Overlay.

Step 4: Verification Query

  • Query: Rerun the exact same query (king -> power) on the modified Overlay.
  • Result: Path "king -> power" found.
  • Final Cost: 1.4879
    • Final Cost Breakdown: inv_m0=0.9102, vad=0.0000, rel=0.6000, overlay=-0.0223

Step 5: Final Analysis

  • Observation: The cost of the direct path king -> power changed from 1.5102 to 1.4879.
  • Analysis: The penalty delta of +0.01 was applied to this edge.
  • However, the overlay_contrib is normalized: raw_delta / sqrt(degree_out * degree_in).
  • Degrees: out(king)=5, in(power)=2. Denominator = 3.1623.
  • The final overlay contribution was -0.0223, which was not enough to change the outcome.

Conclusion: The model did not change its path. This reveals the critical importance of the degree_normalization in the cost function and shows that learning is non-linear. A small penalty on a highly connected edge can be insignificant. This is a key insight into the model’s behavior.
